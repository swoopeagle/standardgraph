# Crosswalk Quality Scoring — Completion Report (2026-07-18)

## Summary

StandardGraph's crosswalk graph went from **partially scored to 100% quality-scored** over
this work cycle. **All 208,442 crosswalks** now carry a 1–5 LLM quality score, shipped
end-to-end across PyPI, HuggingFace, and the hosted MCP endpoint as **v1.6.0 → v1.6.1**.

Before this cycle, quality scores existed only on a subset of edges (AP/IB source rows,
earlier Sonnet waves). After it, every mapping the tool can return — direct country-to-country
edges and hub-centric mappings across all seven subjects — is graded, and `map_standard`
ranks results by that grade and suppresses the weakest matches.

## Why this matters

Crosswalks are candidate equivalences generated by embedding similarity (cosine matching to a
subject hub, plus the within-family direct layer). Embeddings are fast but imperfect: some
links are genuine equivalences, others share vocabulary without sharing meaning
(e.g. a high-school statistics standard matched to a grade-4 "analyze data" standard). An
unscored graph forces the caller to trust every candidate equally. Scoring separates the
signal from the noise so results are ranked by real alignment quality and the junk is hidden.

## What was scored

| Layer | Edges | When | Flagging policy |
|---|---|---|---|
| Direct-family (country↔country, math) | 88,944 | v1.6.0 (2026-07-17) | flag score ≤2 |
| Hub-centric (state→hub, all subjects) | 41,697 | v1.6.1 (2026-07-18) | flag **score 1 only** |
| (Pre-existing scored: AP/IB, earlier waves) | ~77,801 | prior | — |
| **Total graph** | **208,442 (100%)** | | 37,335 flagged |

The two flagging policies differ deliberately. Direct-family edges were built with grade-delta
filtering within similar curricula, so a score-2 there is usually genuinely weak. Hub-centric
score-2 edges are frequently valid-but-grade-shifted (same skill, adjacent grade), and
`map_standard` already ranks by score — so suppressing all of them would blunt the graph.
Flagging only score-1 hub edges hides the truly unrelated matches while keeping borderline ones
visible but low-ranked.

## Method

**Compute.** Local Ollama fleet, `qwen2.5:14b` on every device for calibration consistency:
- Mac Studio (M1 Max) + Mac mini 2 (M4 Pro) — ~15–17 rows/min each
- IWPC (RTX 3060, CUDA) — ~78 rows/min, roughly 5× a Mac; carried the bulk of the hub run

Scores were written to per-device SQLite shards (`mapping_id, score, rationale`), then merged
into the master DB. The driver script (`scratchpad/fleet_score.py`) runs on the dev machine and
calls each device's Ollama over HTTP; a fixed prompt means the model can only emit a score, so
there is no agentic loop in which it could substitute a heuristic. Shards are idempotent (skip
already-scored ids), and an `--also-skip` flag lets a device skip ids already scored by another
shard, enabling zero-waste rebalancing mid-run.

**Rubric (1–5).** 1 = unrelated · 2 = same subject area, different skill · 3 = related, partial
overlap · 4 = strong match, minor difference · 5 = essentially identical objective. Each score
carries a ≤15-word rationale written into the `notes` column as `[LLM score N/5] <rationale>`,
preserving the original provenance note after a ` · ` separator.

## Results

Hub-centric distribution (41,697 edges): score 1 → 2,824 · 2 → 23,735 · 3 → 6,631 · 4 → 301 ·
5 → 8,206. Weak (1–2) landed at **63.7%** — high, and verified genuine by spot-check: the hub
star-graph really does contain many cross-grade / cross-domain matches that share vocabulary but
not meaning. This is the scoring doing its job.

Non-math hub edges scored about as healthy as math (mean 3.30 vs 3.39), confirming the low
scores reflect real edge quality, not a subject bias.

## Bug found and fixed: prompt pollution

The fleet prompt hardcoded *"score how well two **math** curriculum standards match."* Correct
for the all-math direct-family run, but wrong for the hub run's ~24,000 non-math edges. The
model, told ELA/science/social-studies standards were "math," produced ~2,520 confusing
rationales (e.g. *"these are ELA standards, not math"*).

Measured impact: **scores were not systematically biased** (non-math mean 3.30 ≈ math 3.39);
only the rationale *text* was polluted. Fix: made the prompt subject-neutral
(*"two education curriculum standards"*) and re-scored just the ~2,520 contaminated rows on
IWPC with the corrected prompt (`scratchpad/cleanup_rescore.py`, ~30 min). Post-fix verification
found zero garbage patterns; the only remaining "math" mentions in non-math notes are legitimate
(NGSS science standards that genuinely use mathematics).

**Lesson:** do not hardcode a subject in a prompt intended to run across subjects.

## Incident: hosted endpoint outage (~5h)

During the overnight run, Mac mini 3 went offline for ~5 hours (Tailscale "last seen 5h ago";
`/mcp` returned Cloudflare 530 = origin unreachable). Root cause was a reboot, not sleep (sleep
is already disabled via `pmset`). The serve and tunnel launchd jobs are configured with
`RunAtLoad`/`KeepAlive`, but they are **LaunchAgents**, which only load inside a logged-in GUI
session — so after an unattended reboot with no auto-login, the endpoint stayed down until a
user logged in.

**Recommended fix:** enable automatic login on Mac mini 3 so a reboot restores the session and
both launchd jobs without manual intervention. (Alternatively, promote the jobs to
LaunchDaemons, though they may depend on user context.)
